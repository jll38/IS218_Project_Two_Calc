AAA Testing

AAA (Arrange, Act, Assert) Testing allows for safer code and allows developers to know when code breaks or a problem arises.

Arrange - Set up the test case. Depending on the test, there may be certain objects or special objects needed to test a program or function. These operations should be taken into consideration or executed at the very beginning of the test.

Act - Covers what needs to be tested. This could include calling a function, method, interacting with a web page, or etc. Say an operation returns a number, you could assign num = operation().

Assert - Assert the expected outcomes. This may be as simple as checking a numeric/string value, or checking multiple properties of the system. Continuing from the previous example, say you expect operation() to return 5. For your test, you would have assert num == 5. If operation() runs the way you expect, the test should pass when num is checked, otherwise, the test will fail.

AAA Testing in Calculator

The two functions shown on the figure to the right is a perfect example of how AAA testing works. Here we have the multiply and divide functions from the calculator program.

Arrange - In the multiply test, we first arrange the conditions by creating a calculator object, and adding 1. This is due to the fact that in the calculator class, result is equal to 0 starting out, so if we were to test without setting result to a non-zero number, the result will ALWAYS be zero.

Act - The conditions are set up for the test. We assign a variable to the output of calculator.multiply(5).

Assert - Finally, the output of the multiply method is tested. If the method works as expected, the test will pass.
