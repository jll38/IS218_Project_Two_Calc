OOPs

OOP (Object Oriented Programming), as the name implies, is a programming paradigm that utilizes objects and classes into programming. The main concepts of Object- Oriented Programming are:

  • Classes
  • Objects
  • Polymorphism
  • Encapsulation
  • Inheritance

Glossary

Instantiation -creating a copy of a class which inherits all class variables and methods
Object - An entity that has a state and behavior associated with it.
Class - A user-defined blueprint or prototype from which objects are created
Namespace - A system that has a unique name for each object in python. Python directories, such as the calculator folder containing the calculator py file.
Constructor - used when instantiating an object. Assigns values to data members in the class
Fixture - Functions which run before test functions are ran. Feeds data to tests
Type Hint - Statically indicates the type of a value within python code

Type Cast -Used to specify what type you want a variable to be
Ex: num = float(1) Unit Test - First level of software testing, the smallest testable parts of a software are tested.

Static Method - A method that is bound to the class but not the object of the class. Can't access or modify the class state. We usually use static methods for utility functions. primarily a way to namespace your methods.
Encapsulation - The idea off wrapping data and the methods that work on the data within one unit. This restricts the accessibility of variables and methods, preventing the accidental modification of data. Object's variables can only be changed by an object's method.
Polymorphism - Objects can have many different forms
Abstraction - A blueprint for other classes. Creates a set of methods that must be created in a child class built from the abstract class.
