What is SOLID?

SOLID is an acyronym comprising of the five object-oriented design principles. These principles establish the practices for maintaining and extending a project. Despite what the acronym may imply, it is not an ordered list of actions to do. Rather, it is a guideline for good practices in programing to be followed. These principles apply to any object-oriented programming language

S - Single-Responisbility Principle
O - Open-closed Principle
L - Liskov Substitution Principle
I - Interface Segregation Principle
D - Dependency Inversion Principle

SOLID Principles

Single-Responsibility Principle

The Single-Responsibility Principle states:
"A class should have one and only one reason to change"
This principle simply states that a class or function should only have one and only one responsibility. Due to that fact, there should only be one reason to change it. Say you wanted to make a program that finds the average and maximum/largest value in a list. Rather than having a singular function that returns both values, the average and maximum should be returned by two seperate functions that may be called upon. There are many advantages to following this principle.

  • Easier to localize errors
  • More instances of code reusability
  • Easier testing

Open-Closed Principle

The Open-Closed Principle states:
"Objects or entities should be open for extension but closed for modification."
This principle essentially means that a class should be extendable but without modifying the class itself.
